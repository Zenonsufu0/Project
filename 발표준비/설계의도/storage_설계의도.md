# storage.py — 파일 I/O · 무결성 검사

## 역할

`data/raw/` 하위 7개 데이터 파일의 **읽기/쓰기** 및 **문법·참조 무결성 검사**를 담당한다. 프로그램에서 파일 시스템을 직접 건드리는 유일한 지점.

- 기획서 5절 "데이터 파일" 전체 대응
- 기획서 5.5절 "무결성 확인 및 처리" 구현

## 설계 의도

### 1. I/O 경계 단일화
- `main.py`도 서비스 레이어도 `open()`/`Path.read_text()`를 직접 호출하지 않는다.
- 파일 포맷이 변경되거나 DB로 이관할 경우 **이 파일만 바꾸면 된다**.
- 테스트 시 `DataStore`를 목으로 교체하면 전체 도메인 로직을 파일 없이 검증 가능.

### 2. 로드 관대함 vs 무결성 검사 엄격함의 분리
- `load_*()` (students/courses/enrollments 등): 필드 수가 안 맞으면 **조용히 스킵**(`continue`). 크래시 없이 최대한 복구.
- `load_colleges()`: 예외적으로 필드 수 오류 시 **`IntegrityError`를 raise**. 단과대/전공 목록은 회원가입·로그인 전반에 쓰이므로, 오류 행을 조용히 건너뛰면 메뉴에서 사라지는 단과대/전공이 생겨 더 심각한 결함으로 이어지기 때문.
- `validate_integrity()`: 기획서 5.5절에 따라 **모든 위반을 목록으로 수집** → 프로그램 종료.
- 이 분리 덕분에 `load`를 여러 곳에서 불러도 부작용이 없고, 무결성 검사는 시작 시 1번만 수행하면 된다.

### 3. `ensure_defaults()` 기본값 전략
기획서 5.5절 "파일 누락 시 자동 생성" 요구대로:
- `students.txt`, `courses.txt`, `enrollments.txt`, `completed_courses.txt` → **빈 파일**
- `admins.txt` → 기본 관리자 `admin01,Admin@1234,관리자` (기획서 5.2.2)
- `colleges.txt` → 기본 9개 단과대-전공 (기획서 5.2.4)
- `config.txt` → `〈오늘날짜〉,〈오늘날짜〉` (기획서 5.2.5)

샘플 학생이나 강의를 자동 생성하지 **않는다** — 기획서 취지("빈 상태에서 시작")를 해치고 참조무결성을 어긋나게 만드는 원인이 되므로 제거.

## 무결성 검사 구조

`validate_integrity()`는 두 단계로 실행:

### 1단계: 문법 검사 (각 파일별)
- `_check_students_syntax`: 학번 9자리, 비밀번호 규칙, 이름 한글 완성형, 상태 값
- `_check_admins_syntax`: 관리자ID 형식, 비밀번호(`!@#$%^&*` 특수문자), 이름
- `_check_courses_syntax`: 과목코드 4자리, 분반 2자리, 학점 1~6, 요일 MON~FRI, 시간 HH:MM(분 00/30), 정원 ≥1
- `_check_enrollments_syntax`: 4필드, 신청상태 enrolled/cancelled
- `_check_completed_syntax`: 2필드, (학번, 과목코드) 중복 금지
- `_check_colleges_syntax`: 2필드, 빈 필드 금지
- `_check_config_syntax`: 정확히 1행, 날짜 형식, 2000-2099 범위, 종료일 ≥ 시작일

### 2단계: 참조 무결성 (1단계 통과 시에만)
- `enrollments.txt`의 학번 → `students.txt` 존재 확인
- `enrollments.txt`의 과목코드+분반 → `courses.txt` 존재 확인
- `completed_courses.txt`의 학번 → `students.txt` 존재
- `completed_courses.txt`의 과목코드 → `courses.txt`에 존재 (분반 무관)
- `students.txt`의 단과대+전공 → `colleges.txt` 존재

기획서 5.5절 비정상 결과 메시지 형식을 그대로 따른다:
> `!!! 오류: [파일명] [행번호]행 - 문법 형식이 올바르지 않습니다: '[오류 내용]'`
> `!!! 오류: 참조 무결성 위반 - [파일명]에서 존재하지 않는 ...`

## 주요 함수

| 함수 | 역할 |
|---|---|
| `ensure_defaults(current_date)` | 누락 파일 자동 생성 |
| `validate_integrity()` | 문법 + 참조 무결성 검사, 에러 목록 반환 |
| `load_*() / save_*()` | 각 데이터 파일 로드/저장 |
| `_rows(path)` | 공통 CSV 파싱 (빈 줄/BOM 스킵) |
| `_enumerated_rows(path)` | 행번호 포함 제너레이터 (에러 메시지용) |

## 주요 설계 결정

### 쉼표 분리 시 `strip()` 적용하지 않음
- 기획서 5.1절: "행 맨 앞과 맨 뒤의 표준 공백은 허용하지 않는다" → 공백이 있으면 문법 위반이어야 함.
- 기존 코드는 `strip()`으로 조용히 제거했는데, 그러면 문법 검사가 못 잡는다.
- 현재는 `split(",")` 결과를 그대로 사용 → 무결성 검사가 공백 섞인 필드를 정확히 감지.

### `load_colleges()`는 누적 구조
- 기획서 5.2.4: "동일 단과대 이름이 여러 행에 등장할 수 있으며, 이 경우 각 행은 해당 단과대에 속한 서로 다른 전공을 나타낸다."
- 딕셔너리 `{단과대: [전공1, 전공2, ...]}` 구조로 합친다.
- 순서를 보존하기 위해 `list`를 사용 (파일에 등장한 순서 = 메뉴 출력 순서).

### `save_config()`는 `current_date`를 저장하지 않음
- `current_date`는 런타임 값이므로 `reg_start,reg_end`만 기록.

## 비의도적으로 하지 않은 것

- **트랜잭션 / 롤백 없음**: 서비스 레이어가 메모리 상태를 바꾼 뒤 `save_*()` 호출 시 실패하면 일부 파일만 갱신될 수 있다. 기획서가 원자성을 요구하지 않고 과제 범위상 과도한 설계라 판단.
- **파일 잠금 없음**: 동시 실행을 가정하지 않는다 (기획서 "프로그램 실행 중 파일을 직접 편집할 경우 정상 작동을 보장하지 않는다" 명시).
