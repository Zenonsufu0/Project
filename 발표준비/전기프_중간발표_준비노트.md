## 📋 발표자료 구성 체크리스트

### 표지 (첫 페이지)
- [ ] "중간발표자료" 명시
- [ ] 프로젝트 주제명
- [ ] 팀명
- [ ] 팀원 명단
- [ ] **(잠정) 합의된 동료평가 평점** ← 필수 포함

### 1. 팀원별 담당 파트 (표지 직후, 슬라이드 1~2쪽)
- [ ] 분업 내용 + 담당 팀원 이름 명시
  - 기획한 사람 / 구현한 사람 / 검사한 사람이 다르면 **각각 구분해서 명시**
- [ ] 협업 부분은 "협업"으로 명시
- 표 형식 권장

### 2. 기획서 수정사유·경위 (수정판 제출한 팀만)
- [ ] 수정 건별로: 수정 이유 + 문제 발견 경위 요약
- [ ] 구현 후 검사에서 발견한 문제는 **강조 표시**

### 3. 검사 중 주요 오류 검출
- [ ] 검출한 구현 오류 내용
- [ ] 해당 오류를 발견한 테스트케이스(TC)
- 전부 쓸 필요 없음 — 기억나는 중요한 것들만

### 4. 보강사항 ⭐ (감점 축소 핵심)
- [ ] 정시 제출 마감 후에 발견한 "아차!" 사항들:
  - 기획서 문제점 (모순, 누락, 불명확)
  - 구현물 문제점 (미구현, 기획 미부합)
  - 검사보고서 문제점 (검사 누락)
- **쓰여 있으면 → 감점 대폭 축소**
- **쓰여 있지 않고 본인이 먼저 말하면 → 살짝 축소**
- **아무것도 없으면 → 온전히 감점**
- **강사가 못 찾은 문제점을 써두면 → 가산점!**

> 💡 지각제출 기간(4/27 09:00~15:00)에 문제점을 찾으면: 문서 재제출보다 **발표자료에 쓰는 게 더 유리**

### 5. 2차 확장계획 (선택 — 점수 차이 없음)
- 미리 생각해뒀으면 기재, 없으면 생략해도 됨

## ✍️ 발표자료 작성 주의사항

### 분량
- 학생발표 + Q&A 합쳐 **총 20분**
- 학생발표는 **15분 이내** 권장
- Q&A 시간을 줄인다고 유리하지 않음 — 지적에 대한 **항변 기회**가 사라지는 것

### 담당자 명시 (매 페이지마다)
- 각 페이지(또는 항목)마다 담당자 이름 명시
- 예: `{기획/TC: 이찬혁, 구현/검사: 이수현}`
- **명시된 담당자가 발표·답변해야 함**

### 가독성
- 심미성은 점수 무관 — 꾸밀 필요 없음
- 단, 가독성·시인성은 중요 (글꼴, 굵기, 색 대비, 글자 크기 등)
- **핵심 내용은 반드시 텍스트로 써야 함** — 말로만 하면 감점!
- 발표 대본 수준으로 전부 쓰라는 뜻은 아님

### 제출
- **PDF 1개**, 압축 없이, eCampus "팀프로젝트" → "중간발표자료"
- **팀장(또는 작성자를 잠시 팀장 지정)만 제출 가능**

## 🎙️ 발표 당일 준비

### 발표 전 준비
- [ ] 카카오톡 그룹콜 네트워크 확인
- [ ] 이어폰·마이크 작동 및 **울림 문제 점검** (스피커 비추천)
- [ ] 발표자료 화면에 열어두기
- [ ] 제출 문서들 + 코드 빠르게 열 수 있게 준비
- 각자 편한 장소에서 각자 장비로 진행 (학교 안 올 필요 없음)

### 발표 진행 순서
1. 강사가 그룹콜 걸어옴 → **학생들이 먼저 콜 걸지 말 것**
2. 팀원 전원 응답 (참여)
3. 학생 발표
4. 질의응답

> 공지된 시간보다 최대 **20분 지연** 가능 — 기다릴 것  
> 너무 늦으면 콜 대신 **카톡 문자**로 확인

### 발표 진행 규칙
- 분업한 부분은 **직접 수행한 학생이 발표**
- 발표자료 순서대로 진행 (앞뒤 왔다갔다 X)
- 발언 시 **자기 이름 먼저 말하기** (카톡 UI에 표시되지만 목소리 혼동 방지)
- 무단 불참(그룹콜 불응) = **결시 처리**

## ❓ Q&A 단계 대응

### 강사 질문에 답변
- 단순 질문 → 긴장 말고 담담히 답변
- 모르면 **모른다고 솔직히** — 아는 척이 훨씬 더 큰 문제
- 분업한 부분 질문 → 반드시 **담당 학생이 답변**

### 강사 지적에 수긍 or 항변
- 안심하고 항변 가능 — 틀린 항변 자체는 불이익 없음
- 강사가 잘못 지적하는 경우도 있음 → **차분히 논리적으로 설명**
- 단, 문서가 오해하게 써져 있어서 잘못 지적된 경우 → 문서에 감점

### 주요 지적 내용
- 기획 오류
- 구현 불일치
- 검사 누락

## 👥 동료평가 정리

### 평점 기준

| 평점 | 의미 | 감점 |
|------|------|------|
| **3** | 성실, 기여도 양호 (1인분 이상) | 없음 |
| **2** | 다소 부족했으나 같은 등급이 부당하진 않음 | -1.5점 |
| **1** | 같은 등급이면 심히 부당할 정도로 불성실 | -5점 (심각 시 -10점) |
| **0** | 사실상 미참여. F 요망 | -21점 |

> 감점은 학기 총점 100점 기준, 중간·기말 두 번

### 두 가지 경로
1. **개인별 과제 (서브)** — 강사에게만 제출 (팀원 모르게) ← **이미 마감 (4/24)**
2. **발표자료 (메인)** — 팀원 간 합의된 평점, 표지에 명시 → 이게 주로 반영됨

### 발표 중 동료평가 2점 이하 처리 절차
1. 발표·문답 후 강사가 이유 질문
2. 해당 학생에게 항변 기회
3. 강사가 최종 결정

> ⚠️ 2점 이하 있는데 강사가 넘어가려 하면 → **"동료평가 확인해주삼"** 이라고 요청!
