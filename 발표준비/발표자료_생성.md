## 슬라이드 3 — 검사 중 오류 검출 (1) storage 관련

> `{검사 수행: 조서현}`

**슬라이드 제목:** `검사 중 발견·수정된 오류 — storage`

```
TC 5.5-5
  발견: students.txt 문법 오류 시 오류 메시지에 "[오류 내용]" 레이블 형식 누락
  입력: 잘못된 형식의 students.txt 로드
  예상: "!!! 오류: students.txt 문법 오류 — [오류 내용]"
  실제(수정 전): "[오류 내용]" 레이블 없이 내용만 출력
  수정: storage.py 오류 메시지에 레이블 추가

TC 5.5-7 / 5.5-8
  발견: 참조 무결성 위반 메시지에 파일명 · 필드명 미명시
  입력: enrollments.txt에 존재하지 않는 학번 포함
  예상: "!!! 오류: enrollments.txt — 존재하지 않는 학번 포함"
  실제(수정 전): 파일명 없이 일반 오류 메시지만 출력
  수정: 무결성 오류 메시지에 파일명·필드명 명시
```

**말할 것:**
> "TC를 돌리지 않았으면 놓쳤을 오류입니다. 오류 메시지 형식이 기획서 명세와 달랐는데,
> storage.py 파일에서 발견하여 수정했습니다."

[발표자: 조서현]

## 슬라이드 7 — 보강사항 ⭐ (감점 축소 핵심)

> **이 슬라이드는 반드시 텍스트로 작성** — 말로만 하면 감점 축소 효과 없음
> `{구현: 이도현}`

**슬라이드 제목:** `보강사항 — TC 6.8-4/5/6 기이수 처리 확인 다이얼로그`

```
■ 기획서 명세 (6.8절)
  수강신청 중(enrolled)인 과목을 기이수로 추가하려 할 때:
    "수강신청 중입니다. 기이수 처리하시겠습니까? (1: 예 / 0: 아니오) >"
      → 1(예): 기이수 등록 완료 + 해당 수강신청 자동 취소
      → 0(아니오): 기이수 처리 중단

■ 실제 구현
  확인 다이얼로그 없이 수강신청 여부와 무관하게 즉시 기이수 등록 처리됨

■ 영향 TC
  6.8-4: 확인창 표시 미구현
  6.8-5: y 선택 시 수강신청 자동 취소 미구현
  6.8-6: n 선택 시 처리 중단 미구현

■ 원인
  student_service.add_completed()가 enrolled 상태 체크 없이 구현됨
  마감 전 발견했으나 cancel() 호출 추가 시 수강신청 기간 의존성 문제 등
  연쇄 수정 범위가 커서 안전상 코드 수정 보류
```

**말할 것:**
> "기이수 과목 추가 기능에서 수강신청 중인 과목을 기이수로 처리할 때
> 기획서가 명시한 확인 다이얼로그가 구현되지 않은 점을 발견했습니다.
> student_service.py 수정이 필요하며, 수정 범위가 넓어 제출 전 보류했습니다."
